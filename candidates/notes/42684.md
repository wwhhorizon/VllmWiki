## Evidence summary
Triton top-k/top-p kernel (_topk_topp_kernel) computes row pointers as LOGITS + row_id * VOCAB_SIZE → correct only for contiguous logits. Non-contiguous (padded-vocab sliced view) → wrong physical row read → all -inf mask → NaN. FlashInfer path already calls .contiguous(). Fix: add contiguity guard in wrapper.
