## Evidence summary

**现象**：same-name LoRA adapter hot-reload (`load_inplace=true`) 下 prefix-cache block 从旧 adapter 的 KV 数据复用。Reproducer 10/10 或 4/4 确定性。Multiple failure paths 确认（`load_inplace`, `unload+load`, `enforce_eager`, concurrent path）。Multiple bypasses 确认（no-prefix-cache, `cache_salt` change, prompt change, cold restart, unique name）。

**根因**：`vllm/v1/core/kv_cache_utils.py` 中 LoRA block hash 只用 `lora_name`，未包含 `lora_path`、content hash、`lora_int_id` 或 runtime version counter。Runtime load/unload 路径不触发 prefix-cache eviction。

**Related**：#30931 (same-name LoRA prefix-cache collision), #38606 (KV block corruption under rapid LoRA alternation), #41400/#41482 (different aspects of LoRA caching).
