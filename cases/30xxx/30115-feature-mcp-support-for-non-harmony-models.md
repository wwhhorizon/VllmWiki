### 🚀 The feature, motivation and pitch In August after GPTOSS was released, a lot of work was put in to enable MCP tool calling with vLLM. GPTOSS is the first model that supported this. However it is a special case as GPTOSS comes with OpenAI harmony (https://github.com/openai/harmony/) as its own parser, as opposed to most models which come with a chat template (ie Minimax https://huggingface.co/MiniMaxAI/MiniMax-M2/blob/main/chat_template.jinja) which we use via the [_preprocess_chat()](https://github.com/vllm-project/vllm/blob/main/vllm/entrypoints/openai/serving_engine.py#L1072) function. This feature request focuses on MCP support for non harmony models --basically any model that has a chat template.
