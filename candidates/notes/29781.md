## Evidence summary
Under LMCache CPU offload, high concurrency causes repetitive / wrong generations, and debugging points to a specific block ID whose 16-token KV chunk differs from the baseline and from other requests with the same prompt. That is almost textbook KV identity corruption across offload / block reuse boundaries, so it deserves mainline tracking.
