### Issue 正文摘录

### Your current environment ### 🐛 Describe the bug I am reporting a functional issue with the Disaggregated Serving (Prefill/Decode separation) feature in vLLM v0.14.0. The system can successfully initialize the cluster and transfer KV Cache between the Prefiller and Decoder using either MooncakeConnector or P2pNcclConnector. However, the generated content from the Decoder is completely incorrect. Instead of answering the prompt, the model produces incoherent, unrelated, or "garbage" text. Observed Behavior: The Prefiller processes the prompt normally and triggers the KV transfer. The Decoder receives the request and starts generating tokens. The Problem: The actual text generated by the Decoder is unrelated to the input. For example, when asking "1+1=?", the model might output random characters or a completely different topic instead of "2". This suggests that while the KV Cache data is physically transferred, it may be corrupted, misaligned, or incorrectly mapped in the Decoder's memory pool during the v0.14.0 KV-transfer process.
