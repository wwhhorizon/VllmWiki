## Why defer_blocked
kv_cache_dtype="fp8_e5m2" silently corrupts output on Qwen-VL models with default scaling (calculate_kv_scales=False). Failure rate scales with input pixel variance — uniform-color images fail frequently. Directly quant_dtype_semantics + kv_cache_identity: FP8 E5M2 KV cache quantization with default scales produces degenerate output for specific weight distributions. PR #41408 (mentioned, not closes) adds a fail-fast guard but is not a direct fix. Issue open, 5 comments.

## Evidence summary
Issue open. Qwen2-VL-2B/Qwen2.5-VL-3B affected; InternVL3-1B not affected (0% failure). Workarounds: switch to fp8_e4m3 or calculate_kv_scales=True. PR #41408 adds post-load attention guard for Qwen2-VL/Qwen2.5-VL.

## Next action
Track if #41408 or a follow-up lands a proper fix; verify if default scaling behavior changes.
