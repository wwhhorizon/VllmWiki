## Why not promoted

## Evidence summary

**现象**：temperature=0 下相同 prompt 在不同 run 产生完全不同输出。10/10 repro on 3 个独立 trace。三个 finding：00450（5 reqs，r2/r3/r4 分化）、01410（21 concurrent，11/21 分化）、00030（cancel+retry，retry 10/10 分化）。

**假设**：V1 base scheduler 未开启 prefix caching 时，block allocator 无 hash-table identity tracking。已完成/取消的 request 释放 KV block 到 free pool，但 block 内容未清零。subsequent request 从 stale KV data decode。

**硬件相关性**：A10/GDDR6 受影响，A100/HBM2e 不受影响——与 dirty-block-reuse 假设一致（HBM2e 可能有 hw-level 清零或更强的 cache coherency）。

**Community confirmation**（2026-06-04）：`PierpaoloSorbellini` 在 A10 + v0.18.1 + prefix-caching + chunked-prefill 下确认复现，并有 structured output malformation。

New insight：recycled block zeroing 需两个 gate 同时闭环——spec family 声明需要 zeroing，allocator/manager 用 isinstance 覆盖 FullAttentionSpec 子类并把 new block ids 交给 zeroing pipeline。此问题与 #37076 不同（#37076 是 uninitialized-but-registered block race）。
