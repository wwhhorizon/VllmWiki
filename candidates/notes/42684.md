## Evidence summary
`_topk_topp_kernel` computes row pointers as `LOGITS + row_id * VOCAB_SIZE`, so non-contiguous sliced padded-vocab logits can read the wrong physical row, turn a row into all `-inf`, and then produce NaNs through `log_softmax`. The native path lacks the contiguity guard that FlashInfer already uses, which narrows the likely fix shape, but that workaround clue is still not a landed closure.
