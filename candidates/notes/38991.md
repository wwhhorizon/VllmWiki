## Why excluded

Core mechanism is a shared-buffer race condition in runai-model-streamer: yielded tensors are `torch.from_numpy()` views aliasing the streamer buffer. On unified memory platforms (DGX Spark GB10, AGX Thor), `copy_()` from CPU view to CUDA param is async → buffer overwritten before copy completes. Recommended fix: `tensor.clone()` in `weight_utils.py`.

Not a bitwise/deterministic mechanism. Quantization (FP8/NVFP4) is symptom amplifier only — the root cause is concurrency/memory safety. 0 comments, 0 maintainer engagement since 2026-04-04. Platform-specific (unified memory ARM64 only). Per agent_loop rules: excluded from bitwise ledger.

## Evidence summary

Shared numpy buffer → torch.from_numpy view → async cross-device copy → silent weight corruption. `tensor.clone()` detaches from shared buffer and fixes. Bitwise-identical reads confirmed from streamer library. Simple local fix, no upstream PR.
