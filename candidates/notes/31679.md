## Why not promoted
Closed issue with linked bugfix PR #31841, but the current local evidence is still at the “race fixed” summary level. Good enough to keep as a bounded async multimodal lifecycle item, not yet enough for stable promotion.

## Evidence summary
The issue reports that Qwen3-VL crashes only when `--async-scheduling` is enabled, and the linked PR says it fixes a race where the previous iteration's async copy can still be reading the CPU `is_mm_embed` buffer while the current step mutates it. That makes this a state / buffer-lifecycle identity bug in the async multimodal path, adjacent to KV/metadata identity even though the corrupted object is a multimodal CPU-side flag buffer rather than KV itself.
