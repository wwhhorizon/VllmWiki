## Evidence summary

**现象**：`LMCacheMPConnector` 中外部 KV cache key 不包含 LoRA adapter identity。两个不同 LoRA adapter 共享相同 token IDs + `cache_salt` 时解析到同一外部 cache entry → cross-adapter KV hit。K/V tensor 是 adapter-dependent（特别是 targeting attention projections 的 LoRA）。

**根因**：`IPCCacheEngineKey` 由 `model_name` + `world_size` + `worker_id` + `token_ids` + `start/end` + `request_id` + `cache_salt` 构成，缺失 `lora_name` / `lora_int_id` / adapter path / hash。External filesystem key pattern 证实无 LoRA 成分：`Qwen-SEP-Qwen2.5-7B-Instruct@0x01000100@<chunk_hash>@shared-lora-salt.data`。

**社区复现**：Sunt-ing 在 2026-06-02 完整体验证。Unpatched：`adapter_a` found_count=0 → `adapter_b` found_count=3（cross hit）。Patched（fold `lora_name` into key）：`adapter_b` found_count=0（cross hit gone），再次 `adapter_b` found_count=3（same-adapter reuse preserved）。Toggle clean。

**Related**：LMCache/LMCache#2961, LMCache/LMCache#2962, LMCache/LMCache#2878。
